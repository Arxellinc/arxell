# Security Policy

## Supported Versions

Security fixes are prioritized for the `main` branch and the latest release line.

## Reporting a Vulnerability

Please do not open public issues for suspected vulnerabilities.

Instead, report privately with:

- A clear description of the issue
- Steps to reproduce or proof of concept
- Impact assessment
- Any suggested remediation

If repository security advisories are enabled, use GitHub private vulnerability reporting. Otherwise, contact maintainers directly through private channels.

## Response Expectations

- Initial acknowledgment target: within 72 hours
- Triage and severity assessment: as quickly as possible
- Fix and disclosure timeline: depends on severity and exploitability

## Disclosure

We follow coordinated disclosure. After a fix is available (or mitigation is in place), we will share an advisory with affected versions and upgrade guidance.
