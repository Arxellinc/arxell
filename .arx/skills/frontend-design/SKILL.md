This skill guides creation of distinctive, production-grade frontend interfaces that avoid generic "AI slop" aesthetics. Implement real working code with exceptional attention to aesthetic details and creative choices.

## Design Thinking

Before coding, understand the context and commit to a bold aesthetic direction:
- Purpose: what problem the interface solves and who uses it.
- Tone: choose a clear design direction and execute it intentionally.
- Constraints: framework, performance, accessibility, and product context.
- Differentiation: identify the memorable visual idea.

Then implement working code that is:
- Production-grade and functional
- Visually striking and memorable
- Cohesive with a clear point of view
- Meticulously refined in detail

## Frontend Aesthetics Guidelines

Focus on:
- Typography: avoid generic fonts, choose characterful pairings.
- Color and theme: commit to a cohesive palette with strong accents.
- Motion: prioritize meaningful reveal, hover, and scroll interactions.
- Spatial composition: use asymmetry, overlap, negative space, or controlled density.
- Backgrounds and visual details: add depth, texture, and atmosphere.

Avoid generic AI-generated aesthetics such as overused font families, cliched gradients, predictable layouts, and cookie-cutter component patterns.

Interpret creatively and make unexpected choices that feel designed for the specific context.
