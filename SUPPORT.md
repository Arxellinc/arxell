# Support

## Getting Help

Use GitHub Issues for:

- Bug reports
- Feature requests
- Documentation improvements

When filing an issue, include:

- Environment details (OS, app version, runtime details)
- Steps to reproduce
- Expected vs actual behavior
- Relevant logs/screenshots

## Security Issues

Do not file security vulnerabilities publicly. Follow [SECURITY.md](./SECURITY.md).

## Project Scope

This project is under active development. Maintainers may prioritize:

- Reproducible bugs
- Security issues
- Regressions
- PRs aligned with roadmap

## Response Time

Support is best-effort. No guaranteed SLA is provided.
