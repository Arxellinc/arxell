# Architecture Decision Records

Use ADRs to capture boundary, contract, and platform decisions during rebuild.

## Files

- `000-template.md`: template for new ADRs.
- `001-layer-boundaries.md`
- `002-tool-contract.md`
- `003-event-model.md`
- `004-cancellation-model.md`

## Rules

- Create a new ADR when changing contracts, events, or layer boundaries.
- Keep ADRs short and concrete.
- Mark old ADRs as superseded instead of deleting them.
